# GUI 预发布架构说明（2026-03-24）

## 1. 当前预发布主线

当前项目的预发布主线已经收敛为：

```text
GUI
  -> app_services.inference_service
      -> 全图切片推理
      -> IOS 跨片合并
      -> scratch 链连接
      -> DetectionResult 统一对象
      -> scoring.quantify
      -> scoring.wear_score
  -> 可视化输出
      -> 检测叠加图
      -> 缺陷地形图
      -> 评分分解图
      -> 等级卡片
```

GUI 主入口：

- `src/darkfield_defects/viz/app.py`

应用服务层：

- `src/darkfield_defects/app_services/inference_service.py`

统一评分链：

- `src/darkfield_defects/scoring/quantify.py`
- `src/darkfield_defects/scoring/wear_score.py`

## 2. 当前 GUI 的主体功能

当前 GUI 已固定支持以下核心功能：

1. 导入单张镜片图像
2. 调用默认最优模型权重执行全图推理
3. 对跨切片缺陷执行全图后处理合并
4. 生成统一的磨损量化指标
5. 输出统一评分与 A/B/C/D 等级
6. 展示 4 个结果面板：
   - 检测结果图
   - 缺陷地形图
   - 评分过程图
   - 评级卡片

## 3. 当前不属于 GUI 预发布主线的内容

以下内容保留为研究或测试分支，不属于 GUI 预发布主线：

- `scripts/train_*.py` 训练脚本
- SSGD 迁移学习实验
- SFE/NWD 结构实验
- 分割训练实验
- `scripts/run_cnas_eval.py` 第三方测试脚本
- 传统检测 CLI 主线

## 4. 统一后的评分理论

当前评分采用统一的五项主框架：

- `L_center`
- `L_total`（区域加权）
- `N`（区域加权）
- `A`（区域加权）
- `S_scatter`（区域加权）

并采用三分区定义：

- `center`
- `microstructure`
- `edge`

其中：

- 中心区代表主视觉区
- 微结构区代表离焦微结构关键功能区
- 边缘区代表辅助区域

当前评分实现已经不再区分“GUI 评分”和“包内评分”两套逻辑，统一使用 `src/scoring`。

## 5. 当前建议的版本定位

当前代码状态适合作为：

- `v0.1-pre` 预发布内部版本

其特点是：

- GUI 主线清晰
- 评分实现统一
- 自动化测试通过
- 研究分支与产品主线已开始分离

后续继续演进时，建议优先处理：

1. 将 CNAS 测试独立成专用子系统
2. 将研究脚本继续从 GUI 主线剥离
3. 再并行推进识别算法分支与分割算法分支
