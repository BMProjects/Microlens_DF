# 暗场眼镜片缺陷检测中的分割模型选型与集成方案深度研究

## 执行摘要

你的暗场眼镜片缺陷场景（高分辨率、缺陷细长/弯曲/交叉、且常出现大面积重度损伤）已经呈现出明确的“任务形态迁移”：使用边界框（bbox）把长划痕/密集裂纹表达成多个碎片实例，会导致**标注不稳定、跨切片不可合并、评估被碎片化惩罚**；而像素级分割能更自然地表示“缺陷区域场”，并直接服务后续面积/密度/长度等统计分析。与此同时，你现有的 Frangi + 阈值 + 形态学管线已经天然产出“弱分割标签”，这使得以分割为基础的系统升级在工程上更可行：先把弱标签变成可训练的 mask，再通过教师-学生/伪标签迭代提高 mask 质量，最终再把 mask 投影回 bbox 以保持 mAP 可比性。citeturn5search0turn5search3turn8search0turn8search15

在近三年（约 2023–2026）语义/实例分割的主线发展中，**Transformer 分割框架（SegFormer、Mask2Former）**与**现代卷积分割框架（SegNeXt、ConvNeXt 系）**在“准确率—算力—多尺度鲁棒”上形成互补；此外 **高分辨率保持类（HRNet/HRNet-OCR）**在细结构保真方面仍十分有竞争力。citeturn0search0turn0search1turn0search2turn1search2turn0search3

综合“细长结构保真 + 多尺度上下文 + 高分辨率工程可落地 + 弱标签/噪声鲁棒 + 4090D 训练资源”的约束，本报告给出优先级最高的 3–5 个候选模型与明确的集成步骤：

- **首推基线（强性价比）**：SegFormer-B2（MiT-B2）+ 滑窗（mmseg slide）+ thin-structure 损失（Dice/FTL + boundary/clDice 可选）+ mask→bbox 投影，适合以较少工程改动快速把系统“从检测改造成分割驱动”。citeturn0search0turn9search3turn8search0turn6search0turn5search3  
- **高分辨率细结构优先**：HRNet(+OCR) 或 HRNetV2-Seg 作为对照；其持续高分辨率表征对细线/裂纹连通性更友好，常作为“细结构保真”上限参照。citeturn0search19turn0search3turn1search35  
- **强上下文与稳健多尺度**：UPerNet + Swin-T（或 Swin-B）对大范围损伤区与跨尺度上下文建模强，代价是推理更重。citeturn1search1turn3search0turn3search16  
- **现代卷积注意力路线**：SegNeXt（MSCAN）常在细节与效率之间取得很强平衡，适合缺陷纹理细节丰富、且希望推理更轻快的产线系统。citeturn0search2turn0search14  
- **追求上限（实例+语义统一）**：Mask2Former（通常配 Swin/ConvNeXt backbone）作为“高精度上限对照”，更适合在你们数据闭环成熟后冲高精度，工程复杂度最大。citeturn0search1turn0search9turn0search21

最后，报告给出一套“AI 助手可直接照做”的逐步清单：从数据准备（弱 mask 清洗）、训练（loss 组合、采样、伪标签教师-学生）、推理（滑窗 overlap、重叠区域软融合、连通域/骨架化）、到评估（mIoU/Dice/边界 F1/骨架指标 + mask→bbox AP + 按镜片分组切分避免泄漏）。

## 近三年分割模型版图

近三年语义/实例分割的技术栈可以按“编码器范式 + 解码/查询范式”理解：一类是 **轻解码 + 多尺度编码器**（SegFormer、SegNeXt）；另一类是 **查询/掩码注意力统一框架**（Mask2Former），以及“经典但仍强”的高分辨率分支（HRNet-OCR）与编码器-解码器（DeepLabv3+、U-Net 及其增强版）。citeturn0search0turn0search1turn0search3turn2search3turn5search0

### Transformer 语义分割家族

- **SegFormer（MiT 编码器 + All-MLP 轻解码）**：核心点在于分层 Transformer 输出多尺度特征，且**不使用位置编码**，从而在训练/测试分辨率不一致时更稳定；并提供 B0–B5 多尺度模型族，便于按算力选择。citeturn0search0turn9search3turn9search35  
- **Segmenter（ViT + 线性/Mask Transformer 解码）**：强调在第一层起就建模全局上下文，适合中等规模数据微调，但往往更吃分辨率/显存（取决于 patch size 与 token 数）。citeturn5search1turn5search5  
- **Swin 作为骨干的分割（UPerNet+Swin、Swin-UNet）**：Swin 采用移位窗口的层级结构，对高分辨率更高效、可线性扩展到大图；UPerNet 解码大范围上下文强，Swin-UNet 则以 U 形结构强调细节恢复。citeturn1search1turn3search0turn3search1  

### 查询/掩码注意力统一家族

- **Mask2Former**：以“masked attention”把 cross-attention 限制在预测 mask 区域内，实现语义/实例/全景统一架构，并在多基准上达到非常强的结果；工程上代价是训练和推理更重，且对部署（ONNX/TensorRT）要求更高。citeturn0search1turn0search9turn4search23  

### 现代卷积与高分辨率家族

- **SegNeXt（卷积注意力 MSCAN）**：主张“卷积注意力”比 Transformer 自注意力更高效地编码上下文，在多个基准上取得强性能与较好效率，作为工业分割常用“高性价比”路线。citeturn0search2turn0search6turn0search14  
- **ConvNeXt 作为 backbone 的分割**：ConvNeXt 论文指出其作为纯卷积骨干可在多任务上与 Transformer 竞争，并在 COCO/ADE20K 等任务上表现强；工程上通常与 UPerNet/Mask2Former 等解码器组合。citeturn1search2turn1search6  
- **HRNet/HRNet-OCR**：HRNet 用多分辨率并行分支保持高分辨率语义；OCR 进一步通过对象-上下文聚合改善语义一致性，常被用于需要细结构保真的场景（例如线/边界类目标）。citeturn0search35turn0search3turn0search19  
- **DeepLabv3+**：ASPP 捕获多尺度上下文，decoder 细化边界，是经典且仍非常实用的基线，尤其适合对“边界锐利度”有要求的工业分割。citeturn2search3turn2search7  
- **U-Net 与 U-Net++ / UNet3+**：U-Net 强调通过编码器-解码器与跳连实现精确定位，且在小数据下可依赖强增强；UNet++/UNet3+则通过更密集或全尺度连接与深监督改善融合与边界细节。citeturn5search0turn3search2turn3search3  

## 候选模型在暗场镜片缺陷场景的适配性分析

你的暗场缺陷具有三类结构特征：  
1）**细小斑点**（spot/pit 类，像素尺度小、强不均衡）；2）**细长曲线**（scratch/crack 类，连通性与拓扑更重要）；3）**大面积损伤区**（critical 区域，强上下文依赖、且会抬高整体亮度）。因此分割模型需要同时兼顾“小目标敏感性、细结构保真、跨尺度上下文”。citeturn6search0turn6search1turn0search0turn1search1

## 管线集成与工程适配步骤

这一节给出“把分割接入现有暗场缺陷管线”的**具体工程改造**：输入切片/重叠融合、多尺度推理、训练采样、后处理与 mask→bbox 投影。整体原则是：**训练仍以 640 patch 为主**（保护小 spot 尺度），但推理改为“原图滑窗 + overlap + 融合”，并在原图层面做结构化后处理（连通域/骨架/多边形/折线）。citeturn8search0turn8search15turn2search2turn2search10

### 系统流水线示意

```mermaid
flowchart LR
A[原图 4096x3000 ROI] --> B[滑窗切片\ncrop_size + stride + overlap]
B --> C[语义分割模型\n输出 P(class|pixel)]
C --> D[重叠区域融合\nsoft-voting/加权平均]
D --> E[阈值化 + 形态学清理\n开运算/小连通域过滤]
E --> F[结构化提取\n连通域/骨架化/多边形简化/折线]
F --> G[统计分析\n面积占比/长度/密度/分布热图]
F --> H[mask→bbox 投影\n用于 mAP 可比评估]
H --> I[与检测系统对照/集成\n(可选) 联合输出]
```

### 输入切片、滑窗与重叠融合

在高分辨率工业图像上，**滑窗推理是“必须品”而非可选项**：MMSegmentation 明确提供 `test_cfg.mode='slide'` 并要求设置 `crop_size` 与 `stride`，用于重叠滑窗推理；其源码也明确实现了“带重叠的滑窗推理”。citeturn8search0turn8search15

建议你对每个候选模型统一采用如下推理策略（用于消融）：

- `crop_size`：与训练 patch 一致（例如 640×640）或更大（如 768/896）  
- `stride`：`stride = crop_size * (1 - overlap)`，overlap 先从 0.25–0.33 试起  
- 融合：对重叠区域做 **logit 级别**加权平均（比硬拼接更平滑），再做 softmax/阈值化  
- 额外加速：仅对镜片有效 ROI 做滑窗（减少背景）  

如果你们更习惯 SAHI 那条生态，可以把它用于检测/实例分割；SAHI 的核心是“切片→推理→拼接”，并在实践中常用于大图/小目标场景。citeturn2search2turn2search10

### 面向细长缺陷的训练采样策略

细长划痕/裂纹在“随机均匀裁剪”下很容易被裁成碎片，建议采用“结构感知采样”：

- **正样本引导裁剪**：以 Frangi/弱 mask 的连通域为中心做裁剪（确保 patch 内包含完整划痕片段+上下文）  
- **多尺度 patch 体系**：训练时仍以 640 为主，但可混入少量更大 patch（如 768）用于增强上下文，并通过随机缩放抖动保持 spot 可见性  
- **难例采样**：对“整体偏亮、密集大缺陷”的镜片增加采样权重（这些样本最容易导致误分割噪声）  

### loss 设计、辅助任务与后处理

你的缺陷是典型“强类不均衡 + 薄结构拓扑敏感”。因此 loss 建议从“区域 + 边界 + 拓扑”三层组合，不需要一上来全堆，但要有明确的演进路径：

- **区域主损失（默认基线）**：BCE/CE + Dice。U-Net 论文强调强数据增强可在少量标注下有效训练，也暗示对小数据时需要“强约束 + 强增强”。citeturn5search0  
- **小结构不均衡增强**：Focal Tversky Loss 专门为“小结构/不均衡”提出，在医疗小病灶中用于平衡 Precision/Recall，可迁移到 spot/细裂纹。citeturn5search3  
- **IoU 对齐增强**：Lovász-Softmax 可直接优化 mIoU 的 surrogate，在前景稀疏时常优于单纯 CE。citeturn5search2turn5search6  
- **边界增强（可作为二阶段 finetune）**：Boundary loss 以轮廓距离为目标，适合高度不平衡问题并可与区域损失相加。citeturn6search1turn6search13  
- **连通性/拓扑增强（薄结构关键项）**：clDice 以 mask 与其骨架的交集定义拓扑相似度，并给出可微的 soft-clDice，用于训练时保持连通性。citeturn6search0turn6search4  

推荐的演进顺序（更稳）：  
1）BCE/CE + Dice 作为所有模型的共同起点；  
2）若 spot 漏检或细裂纹断裂明显，上 Focal Tversky；  
3）若边界毛刺/过粗，上 Boundary loss（后期加权微调）；  
4）若划痕连通性（跨窗断裂）仍差，上 clDice（与 Dice 组合）。citeturn5search3turn6search1turn6search0  

后处理建议与你现有传统算法逻辑一致（这是优势）：

- 置信度阈值化（可按类不同阈值）  
- 小连通域过滤（min_area）  
- 形态学开运算/闭运算去噪、补洞  
- skeletonization（骨架化）→ 为长度/连通性统计服务  
- 多边形简化 / 折线抽取（为后续“长划痕合并/贯穿统计”服务）

### mask→bbox 投影以保持 mAP 可比性

你当前目标仍需要与检测 mAP 对齐（或沿用现有评估脚本），因此必须提供“从 mask 得到 bbox 的标准化流程”。建议固定如下规则，确保每次实验可比：

1）对每个类别输出概率图 `P_c(x,y)`  
2）阈值化：`M_c = P_c > t_c`  
3）形态学清理 + 连通域：得到若干 component  
4）对每个 component：输出 bbox = 外接矩形，score = component 内 `mean(P_c)` 或 `max(P_c)`  
5）跨窗合并：在原图坐标系下对 bbox 做 WBF（可选）以减少重复框；WBF 原论文提供了融合不同预测框的思路。citeturn8search3turn8search22  

这样你就能在同一套 mAP 脚本上比较：“YOLO 检测框” vs “Seg 模型投影框”。

## 训练与弱监督方案

你的一个独特优势是：已经有 Frangi/阈值/形态学生成的“粗分割结果”。这非常适合做 **弱监督分割**（weak labels）与 **教师-学生伪标签迭代**：第一轮用弱 mask 训练，第二轮用分割模型预测反哺清洗弱 mask，并逐步提高标签质量。U-Net 原论文也明确强调通过强增强和训练策略可在少量标注下有效训练，这与弱监督场景高度契合。citeturn5search0turn5search24

### 预训练与骨干选择

分割 backbone 建议分三类：

- **MiT（SegFormer 系）**：作为 SegFormer 原生编码器，适合你的主基线与 B2 首选。citeturn0search0turn9search35  
- **Swin（UPerNet/Mask2Former 常用）**：对高分辨率友好，适合大范围损伤上下文与更高精度对照。citeturn1search1turn1search5  
- **ConvNeXt（UPerNet/Mask2Former 常见）**：作为现代卷积骨干与 Transformer 竞争，并在多任务上表现强。citeturn1search2turn1search6  

自监督预训练（非常适合你有大量未标注 patch 的情况）：

- **MAE**：通过高比例遮挡重建学习表征，论文给出可扩展的自监督方案与强迁移表现。citeturn4search1turn4search5  
- **DINO / DINOv2**：DINO 论文观察到自监督 ViT 的特征中会“显式包含语义分割相关信息”，DINOv2 则以更大规模无监督学习更鲁棒特征；这对暗场域迁移（照明变化、镀膜差异）尤其值得尝试。citeturn4search0turn4search2turn4search10  

### 弱 mask 清洗与教师-学生细化

建议把“弱标签质量”从一开始就显式建模，而不是把 Frangi mask 当真值：

- **清洗启发式（强烈建议固定成代码，便于复现）**  
  - 去除极小连通域与孤立噪点（对应 spot 的下限要谨慎）  
  - 对细长结构做轻度闭运算以补断裂  
  - 对极大面积“整片高亮”做上限截断（避免把整体亮度抬升误当缺陷）  

- **置信度加权监督**  
  - 对“弱 mask 可信区域”（例如 Frangi 高响应骨架附近）权重更高  
  - 对“弱 mask 边缘/不确定区域”权重更低  
  这类思想与不均衡/边界敏感的 loss 设计相辅相成。citeturn6search1turn6search0  

- **教师-学生（EMA teacher）**  
  1）学生模型用清洗后的弱 mask 训练；  
  2）用 teacher（学生 EMA）在全量数据上推理得到 soft mask；  
  3）对 teacher 预测做再清洗，替换部分弱标签；  
  4）重复 1–3 两轮即可，避免误差自强化。  

### 增强策略与训练配方

暗场图像的显著变化主要来自：曝光/增益、噪声、散射背景变化。增强应偏向“保结构、扰光照”，避免破坏细线几何：

- 光照/对比度：小幅 RandomBrightnessContrast  
- 噪声：Gaussian noise（模拟传感器噪声）  
- 轻微模糊：模拟轻微失焦/散射  
- 几何：旋转/翻转（暗场方向性若明显，则需小心；可先消融）  
- 不建议：过强的 elastic 若会扭曲真实划痕拓扑，应谨慎，仅在证实提升后启用

训练工程建议（适配 4090D）：

- mixed precision（AMP）  
- batch size：以显存能承受的最大值为宜；薄结构任务更依赖稳定梯度，可用梯度累积  
- lr schedule：cosine + warmup（常规稳定做法）  
- epochs：先用较短跑通（50–100），再对最优模型做长训（150–300）  

## 评估协议与指标体系

你要同时评估三层目标：  
1）分割质量（像素层）；2）薄结构质量（边界/连通性层）；3）与检测 mAP 的可比性（mask→bbox）。COCO 指标体系的 AP@[.50:.95] 强烈依赖定位精度，你们此前检测 mAP@0.5 与 mAP@0.5:0.95 的差距就说明“几何表达稳定性”很关键；分割可以从根上改善这一点，但评估必须一致且防泄漏。citeturn8search3turn5search2turn6search0

### 必须固定的验证协议

- **按镜片/原图分组切分（group split）**：同一镜片（原图）切出的 patch 不允许跨 train/val/test，否则会发生信息泄漏与指标失真。  
- **推理必须在原图坐标系合并后评估**：否则长划痕跨窗被拆碎会被过度惩罚（你们已经观察到这一点）。  

### 建议输出的指标集合

像素层（语义分割基础）  
- mIoU、Dice（对稀疏前景更敏感）citeturn5search0  

边界/薄结构层（暗场长划痕最重要）  
- boundary F1（可用边界提取后距离阈值统计）  
- clDice / skeleton-IoU（强调连通性）citeturn6search0turn6search4  
- 断裂率：骨架连通分量数、最大连通长度占比（工程指标）

对象层/镜片层（服务分级与统计）  
- per-defect area recall：缺陷区域覆盖率、最大连通域面积  
- per-lens area coverage：总缺陷面积占比、密度热力图统计  

与检测可比（mask→bbox）  
- 按你们现有检测评估脚本算 bbox mAP@0.5、mAP@0.5:0.95  
- 同时报“每条长划痕聚合后”的对象级召回（可选，但强烈建议作为业务指标）

## 优先实验清单与资源估算

### 推荐的实现短名单与取舍

下面给出 4 个建议你“优先实现对比”的模型组合（其中前 3 个往往足以确定方向；第 4 个用于冲上限）：

1）**SegFormer-B2（主基线）**：速度/精度/工程落地平衡最佳，且对分辨率变化较稳健；适合作为你们分割主干。citeturn0search0turn9search35turn9search0  
2）**SegNeXt-T/S（效率对照）**：如果你希望推理更轻，同时保持细节能力，这是性价比很高的“工业可部署路线”。citeturn0search2turn0search14  
3）**HRNet(+OCR)（细结构对照）**：用于验证“高分辨率保持”对细线连通性的收益上限，为 loss/后处理设计提供参照。citeturn0search19turn0search3  
4）**Mask2Former（上限对照）**：如果前三者已接近上限但仍不足，再引入 Mask2Former 做上限冲刺。citeturn0search1turn0search21  

### 实验矩阵（四周内可跑完的版本）

| 优先级 | 实验 | 关键变量 | 预期收益性质 | 成功标准 |
|---|---|---|---|---|
| P0 | SegFormer-B2 基线跑通 | loss=BCE+Dice；slide 推理 | 建立可复现基线 | 分割可视化合理 + 指标稳定 |
| P0 | 统一滑窗与融合策略 | overlap=0/0.25/0.33；logit融合 | 长划痕碎片化显著下降 | 长划痕跨窗断裂率下降 |
| P1 | 不均衡/小结构 loss | +FTL 或 +Lovasz | spot/细裂纹召回提升 | spot 类 Dice/Recall 提升 |
| P1 | 边界/拓扑增强 | +Boundary 或 +clDice | 细线连通性提升 | clDice↑，断裂率↓ |
| P1 | backbone 对照 | SegFormer-B2 vs SegNeXt-S vs HRNet-OCR | 找到最适合你们数据分布的主干 | 在相同协议下综合最好 |
| P2 | 自监督预训练 | MAE/DINOv2 初始化 vs ImageNet 初始化 | 跨镜片/跨批次鲁棒性 | 新批次/新照明稳定提升 |

loss 依据：Lovász-Softmax 用于优化 IoU surrogate；Focal Tversky 针对小结构不均衡；Boundary loss 针对高度不平衡边界；clDice 针对细长拓扑连通性。citeturn5search2turn5search3turn6search1turn6search0  

### 粗略资源估算（以 4090D 为参考）

由于你未给出精确 GPU 数、I/O、epoch、数据量（mask 数量与类别数），以下为保守估算区间（以 640 patch、AMP 为前提）：

- SegFormer-B2：单次训练（100–200 epoch）约 4–12 GPU-hours；滑窗推理在 4096×3000 原图上取决于 overlap 和 crop_size。citeturn8search0turn9search3

### SAM 辅助标注与交互式精修（提高弱 mask 质量的关键）

建议把 SAM 作为“标注加速器/弱标签修正器”，而不是直接用 SAM 做暗场缺陷推理模型：

- SAM 提供 promptable segmentation，并发布 SA-1B 等大规模数据与模型；更适合交互式标注，而非你这种“缺陷形态高度领域化”的自动分割主干。citeturn1search3turn1search19  
- CVAT 官方文档提供 serverless 方式调用 SAM 进行对象分割，并且 CVAT 的 serverless tutorial 明确列出内置 SAM interactor。citeturn7search12turn7search1  
- SAM2 作为后续可选工具，如果你们做视频追踪/多帧一致性标注会更有价值，但核心缺陷识别仍建议用专用分割网络。citeturn7search2turn7search19

- 建立数据协议

- 加入 mask→bbox 投影与 mAP 输出（保持与你现有检测评估可比）

## 开放假设与风险

以下事项用户未明确或在不同产线条件下可能显著变化，本报告将其视为未知；在启动实验前建议逐项冻结，否则难以定义“成功”：

- 目标类别体系：是否需要区分 spot/scratch/critical 三类像素级标签，还是只要 defect vs background  
- 弱 mask 的类别对应规则：Frangi/阈值/形态学输出是否能稳定分出多类（还是需要后续再分类）  
- 推理延迟目标：单镜片允许的最大推理时延、是否允许滑窗 overlap 与多尺度推理  
- 部署形态：是否必须 TensorRT/ONNX 部署；是否需要边缘端；是否允许 server GPU  
- 可用未标注数据规模：自监督预训练（MAE/DINOv2）能否利用大量未标注 patch（这对鲁棒性影响很大）citeturn4search1turn4search2  
- 镜片有效 ROI 与遮挡：边缘高亮、夹具、反射条纹是否会被纳入缺陷类（需定义）  
- 评估目标：是否坚持 mAP@0.5 作为唯一验收，还是引入“镜片级面积覆盖/报废判定”作为主 KPI（对重度损伤样本非常关键）

一个最重要的风险提示：SAM/SAM2 更适合做交互标注与弱标签精修，不建议直接作为暗场缺陷自动分割主模型；其表现会随图像域变化而波动，文献也强调在不同成像域上需谨慎评估。citeturn1search39turn1search3turn7search2  

另外，模型越“强”（如 Mask2Former），对数据闭环与部署复杂度要求越高；建议先用 SegFormer-B2/SegNeXt/HRNet 定位主要收益来源，再决定是否投入 Mask2Former 冲上限。citeturn0search1turn9search0turn0search14
