## 背景：为什么需要改进

### 当前基线性能

| 类别 | AP@0.5 | AP@0.5:0.95 | 说明 |
|------|--------|-------------|------|
| scratch | **0.4525** | 0.2683 | 细长划痕，**最大短板** |
| spot | 0.8180 | 0.6127 | 点状斑点，表现良好 |
| critical | 0.7589 | 0.4863 | 大面积损伤，表现良好 |
| **mAP** | **0.6765** | 0.4558 | CNAS 通过（≥60%）|

scratch AP 显著低于其他类别的物理根因已在 Phase 2C 日志中确认：

**根因 1：细长 bbox 的 IoU 天然不稳定**

```
示例：一条 200×4 px 的划痕框被向下平移 4 px
  pred:   (10, 10, 210, 14)   面积 = 800 px²
  target: (10, 14, 210, 18)   面积 = 800 px²
  交集面积 = 0 px²（两框完全不重叠）
  IoU = 0.0  → 在 IoU@0.5 评估下，这是一个完全错误的预测

  实际上：预测框与真框的中心偏移只有 4 px，形状完全一致
  这对 spot（50×50 框）不是问题，但对 scratch 是灾难性的
```

**根因 2：切片边界截断长划痕导致弱监督信号**

```
一条贯穿全图的长划痕（长度 ~3000 px）：
  被切成 6~8 个碎片框（每个约 480 px）
  边界碎片（仅露出 20~50 px）在 Frangi 响应图中极弱
  → 模型学习的是"短碎片检测"而非"完整划痕"
  → 导致对真实完整划痕的定位偏差
```

## 改进策略总览

参考 ChatGPT 设计方案（`doc/ChatGPT-眼镜镜片缺陷检测模型.md`）和 GSDNet 论文，确定三轨并行改进策略：

```
Track A (Phase 1): 公开数据预训练
  SSGD 玻璃表面缺陷 → 预训练 YOLOv12m → 私有数据微调
  目标：让模型从更多玻璃缺陷图像中学习划痕形态，改善初始权重

Track B (Phase 2): GSDNet 算法模块集成
  SFE (方向性特征提取) + NWD (Wasserstein 框回归损失)
  目标：直接解决细长目标检测的两个核心问题

Track C (Phase 3): 像素级分割升级
  MSD 数据集预训练 LightUNet → 私有弱标签微调
  目标：为未来 mask 量化（骨架长度、真实面积）铺路
```

三轨互相独立，可并行推进。Phase 4 集成三轨最优成果。

## Phase 1：SSGD 检测预训练

### 策略原理

"公开数据预训练 → 私有数据微调" 是迁移学习的经典范式。其核心假设：
- SSGD 的玻璃划痕与暗场镜片划痕在低层特征上具有相似性（边缘纹理、方向性）
- 在更多（相对于 247 张私有图）且有标注的玻璃缺陷图像上预训练，能让骨干网络学习到更好的划痕特征表示

**训练流程**：

```
COCO 预训练权重 (yolo12m.pt)
    │  epochs=100, lr=0.01, 彩色增强（SSGD 是 RGB）
    ▼
SSGD 预训练权重 (ssgd_trained/weights/best.pt)
    │  epochs=80, lr=0.002（低 LR 保护预训练特征）
    │  warmup=5, 灰度增强（私有数据是单通道）
    ▼
私有数据微调 (private_finetuned/weights/best.pt)
    │
    ▼ python scripts/compare_experiments.py
CNAS 对比评估
```

**关键设计决策**：

| 参数 | SSGD 预训练 | 私有微调 | 理由 |
|------|------------|---------|------|
| `lr0` | 0.01 | 0.002 | 微调时保留已学习特征 |
| `hsv_h/s` | 0.015/0.7 | 0.0/0.0 | SSGD 彩色；私有灰度 |
| `warmup_epochs` | 3 | 5 | 微调时更长 warmup |
| `box` loss | 7.5（默认） | 10.0 | 加强定位精度 |
| `freeze` | 无 | 可选 10 | 如果 mAP 回退则冻结 backbone |

**判断逻辑**（执行后根据结果决策）：

```
if scratch_AP > 0.4525:
    SSGD 预训练有效
    if mAP >= 0.6765:
        全参微调成功，继续 → Phase 4
    else:
        mAP 回退，尝试 freeze=10 变体
        if freeze10 版 scratch_AP > 0.4525:
            使用 freeze10 权重
else:
    SSGD 域差过大，放弃 Track A
    → 重点转向 Track B（GSDNet 模块）
```

- `scripts/train_ssgd_finetune.py` — Phase 1.2（支持 `--freeze 10` 变体）

### 论文背景

**论文**：*A lightweight and robust detection network for diverse glass surface defects via scale- and shape-aware feature extraction*

### Phase 2.1：LODConv — 方向约束可变形卷积

**LODConv（Linear Offset Deformable Conv）**是 SFE 的核心算子。

**标准卷积 vs LODConv 的区别**：

```
标准 3×3 卷积：
  采样 9 个固定位置（以中心对称的 3×3 格）
  对各方向特征权重相同
  → 对细长线状结构感知不足

LODConv (morph=0, x 方向)：
  核形状：横向 k×1 条状
  偏移约束：沿 y 方向学习累积偏移
  → 核可以弯曲/倾斜，自适应跟随划痕走向
  → 捕获 x 方向（水平）的形态特征

LODConv (morph=1, y 方向)：
  核形状：纵向 1×k 条状
  偏移约束：沿 x 方向学习累积偏移
  → 捕获 y 方向（垂直）的形态特征
```

**偏移机制（迭代累积）**：

```python
# 与标准可变形卷积不同，LODConv 的偏移是"迭代累积"的
# 中心点固定不动，向两侧累积偏移，形成连续弯曲的采样轨迹

y_offset_new_[center] = 0  # 中心点偏移为 0
for index in range(1, center + 1):
    y_offset_new_[center + index] = (
        y_offset_new_[center + index - 1] + y_offset_[center + index]
    )  # 向右累积
    y_offset_new_[center - index] = (
        y_offset_new_[center - index + 1] + y_offset_[center - index]
    )  # 向左累积

# 物理含义：采样核会形成一条连续的曲线，而非 k 个独立移动的点
# 这使得核能够"跟着"划痕弯曲，而不是在 k 个不相干的位置采样
```

### Phase 2.2：SFEModule — 4 分支并行特征提取

**架构图（以 inc=32, ouc=64 为例）**：

```
输入 x [B, 32, H, W]
    │
    ├──► conv_0 (3×3) → y0 [B, 16, H, W]  ← 局部特征基础
    │         │
    │         ├──► conv_1 (3×3) → y1 [B, 16, H, W]  ← 二阶感受野
    │         ├──► LODConv(morph=0) → y2 [B, 16, H, W]  ← x 方向弯曲采样
    │         └──► LODConv(morph=1) → y3 [B, 16, H, W]  ← y 方向弯曲采样
    │
    └──► Concat [y0, y1, y2, y3] → 输出 [B, 64, H, W]
```

**关键设计**：
- y0 同时作为 4 个分支的共享输入（节省参数）
- 4 分支并行（不是串行），各捕获不同维度的特征
- Concat（不是 Add），保留各分支的独立信息

### Phase 2.3：集成到 YOLOv12m 架构

**集成策略**（保守方案）：

YOLOv12m 的 backbone 分两部分：
- 前两个块（index 2, 4）：C3k2，提取低层特征（边缘、纹理）
- 后两个块（index 6, 8）：A2C2f，具有 attention 机制，提取高层特征

我们选择：
- **替换 C3k2 → SFEBlock**（index 2, 4）：C3k2 是标准 CSP 块，缺乏方向感知能力，SFEBlock 能在此阶段就捕获划痕方向特征
- **保留 A2C2f 不变**（index 6, 8）：A2C2f 已有 attention 机制，再加 LODConv 可能过参数化
- **head 不变**：head 已经足够，避免增加推理复杂度

**自定义 YAML**：`configs/yolo12m_sfe.yaml`

```yaml
backbone:
  - [-1, 1, Conv, [64, 3, 2]]          # 0
  - [-1, 1, Conv, [128, 3, 2]]         # 1
  - [-1, 2, SFEBlock, [256, False]]    # 2 ← 替换 C3k2
  - [-1, 1, Conv, [256, 3, 2]]         # 3
  - [-1, 2, SFEBlock, [512, False]]    # 4 ← 替换 C3k2
  - [-1, 1, Conv, [512, 3, 2]]         # 5
  - [-1, 4, A2C2f, [512, True, 4]]     # 6 ← 保留 attention
  - [-1, 1, Conv, [1024, 3, 2]]        # 7
  - [-1, 4, A2C2f, [1024, True, 1]]    # 8 ← 保留 attention
# head 与 yolo12m 标准 head 相同
```

**模块注册机制**：ultralytics 的 `parse_model()` 通过 `globals()[module_name]` 查找模块类。将自定义模块注入 `ultralytics.nn.tasks.__dict__` 即可在 YAML 中使用：

**模块文件**：`src/darkfield_defects/ml/sfe_modules.py`

### Phase 2.4：NWD Loss — 基于 Wasserstein 距离的定位损失

#### 原理

将 bounding box 建模为 **2D 高斯分布**：

```
bbox (cx, cy, w, h) → N(μ, Σ)
  μ = (cx, cy)                      ← 中心点为均值
  Σ = diag((w/6)², (h/6)²)          ← 6σ 覆盖 bbox 面积 99.7%
```

两个高斯分布之间的 Wasserstein-2 距离（对角协方差简化）：

```
W²(N₁, N₂) = ||μ₁ - μ₂||² + ||σ₁ - σ₂||²
```

**归一化**（相对于目标框对角线长度）：

```
NWD = exp(-W² / (C × diag²_target))  ∈ (0, 1]
Loss = 1 - NWD
```

#### 为什么 NWD 对 scratch 更好

**数值对比（实测）**：

| 场景 | IoU | NWD loss |
|------|-----|----------|
| scratch 190×4, 中心偏移 4px | **0.0000** | **0.0002** |
| spot 50×50, 中心偏移 5px | 0.6807 | 0.0050 |

scratch 情况：4px 偏移导致 IoU=0（两框完全不重叠），但 NWD 仅损失 0.02%。这是因为：
- 对于 190×4 框，对角线 ≈ 190px，归一化后 4px 偏移微不足道
- IoU 对面积高度敏感（面积 800px² 的框，4px 偏移就能导致零交集）
- NWD 对绝对偏移量相对于框大小的比值敏感，天然适合多尺度

**Hybrid Loss 设计**：

```python
# 直接替换 CIoU 风险高（域差大时可能不稳定）
# 使用 α 混合损失，可以调节 NWD 比重

Hybrid Loss = α × NWD_loss + (1-α) × CIoU_loss
  α = 0.5  → 两者均衡
  α = 1.0  → 纯 NWD
  α = 0.0  → 纯 CIoU（退化为标准训练）

# 实现：monkey-patch ultralytics.utils.loss.BboxLoss
# 无需修改 ultralytics 源码，训练结束后自动恢复
```

**实现文件**：`src/darkfield_defects/ml/nwd_loss.py`

#### 消融实验设计

```

  → 验证方向性特征对 scratch AP 的贡献

实验 B2: YOLOv12m + Hybrid Loss α=0.5（纯损失改变，架构不变）
```

## Phase 3：像素级分割能力

### 策略原理

检测 bbox 是代理测量：
- scratch 长度 ≈ `max(w, h)`（不精确，bbox 往往过大）
- 面积 ≈ `w × h`（严重高估，bbox 大部分是背景）

分割 mask 是直接测量：
- scratch 长度 = 骨架像素数（真实路径长度）
- 面积 = mask 非零像素数（真实缺陷面积）

**现有基础设施**（已有，无需重建）：
- `src/darkfield_defects/ml/models.py`：LightUNet（4 类 bg/scratch/spot/damage，Dice+Focal Loss）
- `src/darkfield_defects/ml/predict.py:150`：`mask_to_detection_result()`，已实现 mask→实例转换（连通域+骨架化+散射强度）
- `src/darkfield_defects/data/dataset.py`：DefectPatchDataset，接受 image_dir + mask_dir（灰度 PNG）

```python

LightUNet(in_channels=1, num_classes=4, base_features=64)
CombinedLoss(
    dice_weight=0.5, focal_weight=0.5,
    class_weights=[0.1, 1.0, 1.0, 0.0]  # damage 在 MSD 中不存在，weight=0
)
# epochs=50, lr=1e-3, patch_size=512, patches_per_image=8
```

### Phase 3.2：私有数据弱标签生成

私有数据没有 pixel-level 标注，只有 YOLO bbox 标注。生成弱标签策略：

```
算法流程：
  1. 运行 ClassicalDetector.compute_candidate_map()
     → Frangi 滤波器响应图（对细长结构有强响应）
  2. 从 YOLO tile 标注 → 全图坐标（tile → fullimage 映射）
     tile_y = row × (tile_size × (1 - overlap))
     tile_x = col × (tile_size × (1 - overlap))
  3. 在每个 GT bbox 区域内：
     - 提取 Frangi 响应 ROI
     - Otsu 自适应阈值 + Frangi 全局阈值取最低
     - 阈值内像素 → defect mask
  4. 如果 Frangi 响应极弱（像素数 < 10）：
     → 用整个 bbox 区域填充（宁可粗糙，不漏标）
  5. GT bbox 外的 Frangi 响应丢弃（可能是 FP）

约束：仅使用 train split（val/test GT 不可修改）
```

**弱标签质量说明**：
- 优点：有 GT bbox 约束，不会出现大面积误标
- 缺点：边界不精确，可能将 bbox 内背景区域也标为缺陷
- 用途：足够用于微调（已有 MSD 预训练的分割基础）

**脚本**：`scripts/generate_weak_masks.py`

## 关键已知问题和风险

### Phase 1 风险

**域差（Domain Gap）**：SSGD 是彩色图（自然光反射），私有数据是灰度图（暗场散射）。低层特征可能差异较大。
- 缓解：`freeze=10` 变体保留 backbone，仅微调检测 head

### Phase 2 风险

**SFEBlock 权重初始化**：从 YAML 构建新模型时，SFEBlock 的 LODConv 权重是随机初始化的。标准层权重不能直接迁移。
- 缓解：训练时先尝试部分迁移（`--transfer-weights`），兼容层的权重迁移，不兼容层（LODConv）从头训练

**NWD 数值稳定性**：`exp(-W²)` 对极小或极大的 bbox 可能数值不稳定。
- 缓解：分母使用 `target_diag.clamp(min=1e-6)` 防止除零

### Phase 3 风险

**弱标签噪声**：Frangi 滤波器对暗场图像的响应不总是准确。弱标签质量低于真实 mask。
- 缓解：仅用于微调（不是从头训练），且 MSD 预训练提供了良好的初始化

## Phase 2 执行结果：GSDNet 模块消融实验

### 关键前提：NWD 数值不稳定问题及修复历史

B2/B3 实验在执行中遭遇了 AMP（自动混合精度）环境下 NWD Loss 产生 NaN 的连锁 bug，总共修复 3 轮。以下详细记录，供后续维护参考。

#### NaN Bug #1：float32 精度不足（第一轮）

**现象**：B2 训练在 epoch 23 附近出现 NaN loss，训练崩溃。
**根因**：`nwd_loss()` 中 Wasserstein 距离计算的中间值（`target_diag`）在 AMP float16 下接近零，导致除零或梯度爆炸。

**修复**（`src/darkfield_defects/ml/nwd_loss.py`）：
```python
# 修复前：1e-6 阈值，float16 下仍可能下溢
target_diag = (...).clamp(min=1e-6)
exponent = (-w2 / (C * target_diag))

# 修复后：显式 float32 + 更保守的 clamp + exponent 限制
pred_boxes = pred_boxes.float()
target_boxes = target_boxes.float()
target_diag = (...).clamp(min=1e-3)  # 更保守
exponent = (-w2 / (C * target_diag)).clamp(min=-20.0, max=0.0)  # 限制指数范围
```

**效果**：第一轮修复后 B2 重启，但在 epoch 17 再次 NaN。

#### NaN Bug #2：梯度仍通过 float16 流动（第二轮）

**现象**：B2 epoch 17 NaN。float32 cast 只修复了 forward pass 的精度，但 PyTorch AMP 的梯度回传仍在 float16 下进行，NWD 的高梯度在 half-precision 反传时溢出。

**修复**（`src/darkfield_defects/ml/nwd_loss.py:HybridBboxLoss.forward`）：
```python
# 将整个 forward 包裹在 autocast(enabled=False) 中，强制全程 float32
with torch.amp.autocast(device_type="cuda", enabled=False):
    pred_bboxes_f = pred_bboxes.float()
    target_bboxes_f = target_bboxes.float()
    target_scores_f = target_scores.float()
    target_scores_sum_f = target_scores_sum.float().clamp(min=1e-6)
    # ... 所有损失计算在此 float32 上下文中执行
```

**效果**：第二轮修复后 B2 重启，但在 epoch 20 再次 NaN。

#### NaN Bug #3：学习率峰值阶段的梯度放大（第三轮，最终修复）

**现象**：B2 epoch 20 NaN。epoch 15-25 是 warmup 结束后的 LR 峰值阶段，NWD loss 梯度幅度显著大于标准 CIoU，在 lr0=0.002 下梯度组合（NWD gradient × peak LR）超出 gradient clip（max_norm=10）的控制范围。

**修复**（`scripts/train_phase2_sfe_nwd.py`）：
```python
# B2 和 B3 实验配置添加降低的 LR
"B2": {
    ...
    "lr0": 0.001,  # 从 0.002 降至 0.001，缓解 NWD 梯度与峰值 LR 的组合
},
"B3": {
    ...
    "lr0": 0.001,  # 同上
},
```

**效果**：B2 在 `autocast(enabled=False)` + `lr0=0.001` 双重修复下，第三次重启后稳定训练至 epoch 32（early stop at patience=25），无 NaN。

**总结**：NWD 数值稳定需要同时满足两个条件：
1. `HybridBboxLoss.forward` 必须全程在 `autocast(enabled=False)` 中执行
2. NWD 实验的 lr0 应比标准训练低（建议 ≤ 0.001）

- 像素标准差：2.4（vs 正常图像约 8.9）
- 对比度极低，暗场散射信号极弱

**根因结论：低对比度域偏移（domain shift），不是 bbox 几何问题。**

这解释了为什么 SFE（方向特征）和 NWD（定位损失）都无效：这些改进针对的是"能看到但定位不准"的情况，而实际问题是"部分图像根本看不到特征"。

## 总体结论与后续建议

### 已达成

### 未达成

- ❌ scratch AP 目标 0.55+：最高值仍为基线 0.4525，所有算法改进均未超越
- ❌ mAP 目标 0.72+：最高值 0.6802（B2）

### 根本障碍

FN 分析揭示：**scratch 漏检的核心障碍是低对比度域偏移，不是定位算法问题**。
当前 top-10 硬负样本图像的像素 std 仅为正常图像的 27%（2.4 vs 8.9），任何基于 bbox 几何的改进（NWD、SFE）都无法解决"图像本身无可辨别特征"这一物理限制。

### 最有潜力的后续方向

1. **数据级对比度增强**（最高优先级）
   - CLAHE（自适应直方图均衡化）预处理：专门针对低对比度图像
   - 目标图像：`output/experiments/phase3b_fn_analysis/hard_negatives.txt`（10 个高漏检率全图）
   - 实现位置：`src/darkfield_defects/preprocessing/` 或训练数据增强管道
   - 预期效果：直接攻击根因，可能有显著改善

2. **重新训练 B3（配置修正版）**（次优先级）
   - 当前 B3 在 NaN fix（lr=0.001）应用前运行，结果不代表最优配置
   - 若需评估 SFE+NWD 的理论上限，应重跑 B3（lr=0.001 + autocast fix）

3. **更长训练 + 更好初始化（B1 优化方向）**
   - B1 失败的主要原因是 SFEBlock 随机初始化与 80 epoch 的组合不足以收敛
   - 增加 epochs（120+）或使用更好的 LODConv 初始化策略可能改善
