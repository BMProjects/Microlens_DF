**日期**：2026-03-21
**阶段目标**：分析 scratch AP 偏低根因，尝试半监督重标注提升至 mAP > 0.70
**前置条件**：Stage3 最优权重已训练完成（mAP@0.5 = 0.6621，`output/training/stage2_cleaned/weights/best.pt`）
**阶段结果**：Stage4 失败（mAP 从 0.6543 跌至 0.4728），回滚到 Stage3，构建独立测试集

## Exp-P2C-01：scratch AP 偏低的物理根因确认

**问题量化**：
Stage3 中，scratch AP@0.5 = 0.414，远低于 spot（0.807）和 critical（0.742）。

**Per-class 详细指标（Stage3 全 val 集）**：

| 类别 | AP@0.5 | AP@0.5:0.95 | Precision | Recall | 物理含义 |
|------|--------|-------------|-----------|--------|---------|
| scratch | 0.4139 | 0.2548 | 0.508 | 0.364 | 线状散射缺陷 |
| spot | 0.8070 | 0.6042 | 0.648 | 0.784 | 点状散射缺陷 |
| critical | 0.7419 | 0.4592 | 0.626 | 0.773 | 密集/大面积缺损区 |
| **均值（mAP）** | **0.6543** | **0.4394** | 0.594 | 0.640 | |

**scratch AP 偏低的多层根因分析**：

**根因 1（结构性）：切片边界截断长划痕**

```
切片参数：stride=480 px（overlap=160 px），tile=640 px
一条贯穿全图的长划痕（长度约 3,000 px）：
  切割成 ceil(3000/480) ≈ 6～8 个碎片框
  各碎片从属不同切片，彼此语义割裂
  边界碎片（划痕仅露出 20～50 px）Frangi 响应极低
  → 模型学习目标为"短碎片检测"，而非"完整划痕检测"
```

**根因 2（数值性）：细长 bbox 对 IoU 评估天然不友好**

```python
# 示例：一条 200×8 px 的细长划痕框
# 预测框相对 GT 平移 4 px（仅 0.6% 偏差）
IoU = (200-4) × (8-0) / (200×8 + 200×8 - 196×8)
     = 1568 / (1600 + 1600 - 1568) = 1568 / 1632 = 0.961  # 平行方向平移

# 但如果是垂直方向平移 4 px：
IoU = (200-0) × (8-4) / (200×8 + 200×8 - 200×4)
     = 800 / (1600 + 1600 - 800) = 800 / 2400 = 0.333  # ！跌破 0.5 阈值
```

即：对于高长宽比 bbox（长宽比 > 10:1），仅 4 px 的垂直偏差即可使 IoU 从 0.96 降至 0.33，远比方形 bbox 敏感。

**根因 3（标注噪声）**：Frangi 弱响应框占 44.3%，大量"无线状结构"框混入训练集，稀释了真实划痕特征。

## Exp-P2C-02：Step4 半监督重标注设计（第一版）

**研究目标**：利用 Stage3 模型预测，修正 GT 中的碎片划痕标注，使 scratch 标注从"短碎片"升级为"完整划痕"形态。

**设计方案（`scripts/step4_scratch_relabel.py` 第一版）**：

```
Step 1：Stage3 模型 conf=0.15 推理所有切片 → 映射到全图坐标系
Step 2：全图 IOS NMS（GT去重 threshold=0.75；Pred激进合并 threshold=0.35）
Step 3：connect_scratches（对 GT 和预测 各自执行划痕连接算法）← 致命设计错误
Step 4：merge_gt_with_predictions（extend 已有链 + add 新增预测框）
Step 5：Frangi 验证新增框（过滤噪声）
Step 6：映射回切片坐标，写入 YOLO 标注文件
```

**执行结果（--split both，同时修改 train 和 val）**：

| 指标 | 数值 |
|------|------|
| GT chains（全图级划痕链）| 6,589 |
| extend（现有链被延长）| 3,545 |
| add（新增独立链）| 3,295 |
| 修改的切片数量 | 10,018 |
| 切片总框数变化 | 90,325 → 77,171（**-14.6%**）|
| val split 标注变化 | 21,986 → 16,958（**-22.9%**）|

## Exp-P2C-03：Stage4 重训练失败实验——完整根因分析

**实验目标**：用 step4 重标注后的数据训练 Stage4，验证半监督重标注策略的有效性，目标 mAP > 0.70。

**训练配置**：Stage3 best.pt 微调，lr0=0.001，box=12.0，epochs=60，patience=20

**结果：早停于 epoch 31，最优 epoch 11**

**量化结果对比（Stage3 vs Stage4）**：

| 指标 | Stage3（当前最优）| Stage4（失败实验）| 变化 | 解读 |
|------|:---:|:---:|:---:|------|
| mAP@0.5 | **0.6543** | 0.4728 | **−27.7%** | 灾难性退化 |
| scratch AP | 0.414 | **0.166** | **−59.9%** | 崩溃 |
| spot AP | 0.807 | 0.797 | −1.2% | 几乎不变 |
| critical AP | 0.742 | 0.456 | −38.5% | 严重退化 |
| Precision | 0.595 | 0.518 | −13% | — |
| Recall | 0.638 | 0.471 | −26% | — |

**混淆矩阵关键证据**：scratch 类 **90% 预测为背景**；spot 几乎不受影响（-1.2%）→ 问题高度 scratch-specific。

**三层根因分析**：

**根因 1（致命，主因）：对 GT 执行了 connect_scratches**

```
原 GT 标注（碎片形态，模型已学习）：
  框 A: (10, 50, 100, 8)    # 100×8 px 短碎片
  框 B: (150, 52, 100, 8)   # 100×8 px 短碎片
  框 C: (310, 51, 100, 8)   # 100×8 px 短碎片

connect_scratches 后（外接矩形）：
  合并框: (10, 50, 400, 10)  # 400×10 px 大外接矩形

Stage3 模型预测（仍然预测短碎片）：
  预测 A': (12, 51, 98, 7)
  与合并框 IoU = (98×7) / (400×10 + 98×7 - 98×7) = 686/4000 = 0.172 < 0.5

→ 预测 A' 被计为 FP；合并框被计为 FN → scratch 全算漏检
→ 这等同于将"检测短碎片"任务强制改为"检测长条"任务
```

**根因 2（重要）：val 集同时被修改（--split both）**

```
val 标注：21,986 → 16,958（减少 22.9%）
Stage3 评估基准被破坏，指标不可与 Stage3 直接比较
val 集修改规则：
  - connect 导致多框合并为一框
  - 原 GT 中模型能预测的短框 → 被替换为模型无法预测的长框
  - val loss 虚假下降（框数少了），掩盖了真实的性能退化
```

**根因 3（结构性）：60 epoch 训练不足以完成任务转变**

```
从"检测短碎片（50-200 px）"到"检测长条（400-3000 px）"是根本性任务变化
微调模式（Stage3 → Stage4）需要先"忘记"原有特征，再学习新特征
从实验观察：ep11 达到峰值（0.4728）后持续退化
→ 需要至少 300+ epoch 从头训练，且需要充分的长条 GT 标注
```

**核心教训（写入项目硬性约束）**：

1. `connect_scratches` **只能用于推理后处理**，永远不修改 GT 标注
2. GT 标注应保持碎片级（与模型的实际学习目标一致）
3. 完整划痕的合并在推理端（SAHI + connect_scratches）完成
4. 半监督重标注**只应处理 train split**，val 保持不变以保证评估基准

## Exp-P2C-04：回滚到 Stage3 + 独立测试集构建

**决策依据**：Stage4 失败，Stage3 是当前最优（mAP@0.6543，独立测试集上 0.6765），回滚并固定。

**回滚操作**：

```bash
# 从备份恢复 Stage3 训练标注状态
cp -r output/tile_dataset/labels/train_pre_step4_backup/. \
      output/tile_dataset/labels/train/

# 验证：应为 92,284 个框（Stage3 训练状态）
# 若为 116,193 则说明 step4 被执行了两次（需确认备份正确）

# 清理 Stage4 权重（避免误用）
rm -rf output/training/stage4_relabel/
```

**独立测试集构建（`output/audit/test_set_v1.json`）**：

**选取标准**：
- 从 val split 选取 20 张代表性图像（共 860 个 640×640 切片）
- **排除** ljj 异型样本（异常光照/特殊镜片形态，不代表典型检测场景）
- 按损伤等级平衡分布：

| 损伤等级 | 图像数量 | 典型缺陷描述 |
|---------|---------|------------|
| 轻度 | 4 | 少量细划痕，spot 稀疏 |
| 中度 A | 6 | 多条划痕，部分 critical 区域 |
| 中度 B | 5 | 密集划痕交叉，中等 critical |
| 重度 | 3 | 大面积 critical，划痕密集 |
| 极重度 | 2 | 大范围磨损，多处 critical |

**验证**：在固定测试集上运行 YOLO native val：

```bash
python scripts/run_cnas_eval.py
# 输出：mAP@0.5 = 0.6765 ✅（超过 60% 目标）
```

**最终独立测试集指标（Stage3 权重，20 张图，860 切片）**：
