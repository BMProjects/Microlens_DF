## 整体设计

从训练时的切片推理扩展到全图推理，解决跨切片划痕的合并问题，并输出全图级别的磨损评分。

```
输入：4096×3000 原始 BMP
    │
    ▼ 预处理流水线（Phase 0-1）
  校正后 PNG（背景减除 + Ring 归一化）
    │
    ▼ SAHI 切片推理（tile=640, overlap=160 px, conf=0.25）
  每切片检测框列表（切片坐标系）
    │
    ▼ 坐标映射到全图坐标系
    │
    ▼ 全图 IOS NMS（ios_thresh=0.35，合并重叠切片碎片）
    │
    ▼ connect_scratches（推理端，合并对齐碎片为完整划痕链）
    │
    ▼ WearScore 计算（5 分量加权）
  输出：全图标注框 + WearScore + 磨损等级
```

## Exp-P3A-01：SAHI 推理参数设计与 IOS NMS 选型

**研究问题**：SAHI（Slicing Aided Hyper Inference）的最优 stride 和 overlap 参数是什么？标准 IoU NMS 是否适合细长划痕框的合并？

**切片参数设计依据**：

```
训练时切片参数：tile=640, stride=480, overlap=160 px（25%）
推理切片保持一致：
  重叠 160 px = 25% overlap
  物理含义：一条跨越两个切片的划痕，在两个切片中各有 ≥160 px 的可见段

覆盖率计算（划痕完整出现概率）：
  假设划痕片段最短为 100 px，切片重叠 160 px
  → 该划痕在至少一个切片中完整出现的概率 > 95%
  → 重叠区内的重复检测由 NMS 去除
```

**IOS NMS 设计理由（相对于标准 IoU NMS）**：

```
标准 IoU NMS 的问题（细长框示例）：
  长框 A：(10, 50, 500, 10)  面积 = 5000 px²
  短框 B：(10, 51, 100, 8)   面积 = 800 px²
  交集面积 = 100×8 = 800 px²
  IoU = 800 / (5000 + 800 - 800) = 800 / 5000 = 0.16
  → IoU < 0.5，两框不会被 NMS 合并 → 重复保留

IOS（Intersection over Smaller）：
  IOS = 交集面积 / min(面积A, 面积B)
      = 800 / 800 = 1.0
  → IOS = 1.0 > 阈值 0.35 → 短框被正确抑制

效果：
  ios_thresh = 0.35 时：短划痕碎片（包含在长框内的）被合并
  背景噪声框（与任何框不重叠）不受影响
```

**SAHI 推理最终参数**：

```python
SAHI 参数：
  tile_size = 640
  overlap_ratio = 0.25        # 160 px overlap
  conf_threshold = 0.25
  iou_threshold = 0.6         # SAHI 内部 NMS

全图后处理 NMS（跨切片合并）：
  GT 去重：ios_threshold = 0.75（保守，只去除几乎完全重叠的框）
  Pred 合并：ios_threshold = 0.35（激进，合并划痕碎片）
```

## Exp-P3A-02：WearScore 磨损评分权重设计

**设计框架（5 分量加权求和）**：

```
WearScore = w₁ × f_center + w₂ × f_density + w₃ × f_critical + w₄ × f_scatter + w₅ × f_total

权重分配：
  w₁ = 0.35  → f_center：中心区（光学中心半径内）缺陷面积占比
  w₂ = 0.25  → f_density：全图缺陷密度（KDE 核密度估计峰值）
  w₃ = 0.20  → f_critical：critical 类缺陷面积占 ROI 比例
  w₄ = 0.10  → f_scatter：缺陷区域背景校正后的散射强度均值
  w₅ = 0.10  → f_total：全类缺陷总框数（归一化）
```

**权重设计依据（视光学原理）**：

| 分量 | 权重 | 依据 |
|------|------|------|
| f_center（0.35）| 最高 | 镜片光学中心对视觉清晰度影响最大；同等尺寸划痕在中心的视觉影响是边缘的 3-5 倍 |
| f_density（0.25）| 次高 | 高密度缺陷（交叉网络）产生漫反射，影响对比敏感度；密度非线性影响视觉 |
| f_critical（0.20）| 第三 | 大面积缺损区直接影响透光率，但位置不确定，权重低于中心区 |
| f_scatter（0.10）| 较低 | 深划痕散射强，夜间眩光风险高，但量化精度受图像标定影响 |
| f_total（0.10）| 最低 | 全面缺陷程度，作为补充项防止单类指标偏置 |

**对数饱和函数（防单点饱和）**：

```python
def sat_log(x, scale=25.0):
    """
    将原始指标 x 映射到 [0, 100] 评分空间
    log1p 曲线：少量缺陷时快速增长，大量缺陷时增长减缓
    防止单一极端缺陷将评分推至满分
    """
    return min(scale * math.log1p(x), 100.0)

# 物理含义对比：
# 线性函数：2 条划痕 → 2 分；100 条划痕 → 100 分（过于惩罚多缺陷）
# log 函数：2 条划痕 → 17 分；10 条划痕 → 40 分；50 条划痕 → 70 分（合理饱和）
```

**等级划分阈值（基于 4 个人工标定典型样本）**：

| 等级 | WearScore 范围 | 解读 | 使用建议 |
|------|-------------|------|---------|
| A | 0 ～ 20 | 轻微磨损，视觉影响可忽略 | 继续使用 |
| B | 20 ～ 45 | 轻度磨损，特定光线下偶有感知 | 注意观察 |
| C | 45 ～ 70 | 中度磨损，对比敏感度下降 | 建议更换 |
| D | ≥ 70 | 严重磨损，强烈影响视觉 | 强烈建议立即更换 |

**注**：等级划分阈值基于初步人工标定，尚未经过临床验证（见开放问题 7.4）。

## 开放研究问题与未来工作

### 7.1 scratch AP 偏低问题（优先级 P0）

**问题量化**：

```
scratch AP@0.5 = 0.4525（测试集）
spot AP@0.5    = 0.8180（测试集）
critical AP@0.5 = 0.7589（测试集）
差距 Δ(scratch - spot) = −0.3655，差距巨大
```

**已确认根因（按影响权重排序）**：

1. **切片边界截断（最主要）**：stride=480, tile=640, overlap=160 px → 一条完整划痕被切成 6-8 碎片，碎片间 IoU 为 0，各自作为独立训练样本
2. **细长 bbox IoU 敏感性**：4 px 垂直偏差 → IoU 下降 33%，评估标准对划痕天然不利
3. **GT 标注噪声**：Frangi 弱响应框占 44.3%，大量无真实线状结构的框参与训练

**待验证改进方案**：

| 方案 | 预期 scratch AP 提升 | 实施难度 | 当前状态 |
|------|-------------------|---------|---------|
| OBB（定向边界框，yolo-obb 模式）| +5-10%（理论）| 需完整重标注 | 未实施 |
| overlap 从 160 px 增至 256 px（40%）| +3-5%（推测）| 需重建数据集 | 未实施 |
| SAHI 推理端 connect_scratches | +2-4% recall（推理端）| 已实现 | 推理端可用 |
| 人工审核 20,263 个 Frangi 弱响应框 | +5-10%（训练信号质量）| 4-8 h 人工 | 工具已就绪 |

**注意**：任何针对 scratch 的改进必须仅作用于 train split，不得修改 val/test 的 GT 标注。

### 7.3 物理标定缺失（P2）

**问题描述**：所有检测指标为像素单位，无法输出物理尺寸（mm）。

| 当前状态 | 目标状态 |
|---------|---------|
| scratch_length = 200 px | scratch_length = 0.82 mm |
| spot_area = 380 px² | spot_area = 0.0064 mm² |

**标定方案**：

1. **标定板法**：在相机光路中放置已知间距（如 0.1 mm）的网格标定板，拍摄一张图像，测量网格点间距（像素数），计算 px/mm 转换系数
2. **镜片边缘法**：利用已知直径的标准镜片（如直径 70 mm），在暗场图像中测量镜片边缘圆弧的像素直径，计算转换系数

**估算现有标定参数**（基于图像尺寸和物理知识）：

```
ROI 直径 ≈ 3,800 px（从 ring_mask 测量）
典型眼镜镜片直径 ≈ 65 mm
px/mm ≈ 3800 / 65 ≈ 58.5 px/mm（粗估，待正式标定）
```

## 项目约束与设计规则（教训汇总）

以下规则源自实验失败教训，具有**硬性约束**地位，未来迭代必须遵守：

### 规则 1：GT 标注完整性保护

```
❌ 禁止：对 GT 标注执行 connect_scratches 或任何跨框合并操作
✅ 允许：在推理后处理阶段（inference pipeline）执行划痕连接

根因：GT 标注形态必须与模型的预测目标一致。
      训练时模型学习的是"短碎片"形态，
      修改 GT 为"长条"形态会导致训练信号与预测分布不匹配，
      造成 scratch 类别崩溃（Stage4 失败实验定量证明：AP -60%）。
```

### 规则 2：val/test 集不可修改

```
❌ 禁止：对 val 或 test split 执行任何标注修改
✅ 允许：对 train split 执行标注清洗、伪标签添加、重标注

根因：val 集是评估基准，一旦修改则不同训练阶段的指标不可直接比较。
      Stage4 失败的原因之一即为 val 集标注减少 22.9%，
      使得 Stage4 评估分数不可与 Stage3 直接比较。
```

### 规则 3：亮度通道必须在原始图上运行

```
❌ 禁止：在 gamma 增强后的图像上运行 brightness_channel
✅ 必须：在原始图（未增强）上运行 brightness_channel

根因：gamma=0.4 增强将噪声底层灰度放大 14 倍，
      在增强图上运行亮度通道会将 21.9% 的像素判定为"高亮"，
      crash 类误报爆炸 3 倍（v2 vs v3 实验定量证明）。
```

### 规则 4：半监督重标注的作用范围

```
✅ 允许：step4_scratch_relabel.py 仅使用 --split train
         只执行 extend（延长已有划痕链）+ add（新增预测框）
         不执行 connect_scratches（GT 连接）

✅ 允许：Frangi 验证作为新增框的质量门槛

❌ 禁止：--split both（同时修改 train 和 val）
❌ 禁止：对 GT 框执行外接矩形合并操作
```

### 规则 5：伪标签推理必须使用 stream 模式

```
❌ 禁止：batch=32 × 8471 tiles 一次性加载（OOM）
✅ 必须：stream=True + 按尺度分批
  640 px → batch=64
  960 px → batch=32
  1280 px → batch=16
```

### 附录 C：数据集标注文件格式

**YOLO 格式（每行一个 bbox）**：

```
<class_id> <x_center> <y_center> <width> <height>
# 所有坐标归一化到 [0, 1]，相对于切片尺寸（640×640）

# 示例（一个 scratch 框）：
0 0.456 0.312 0.823 0.047
# class=0(scratch), 中心=(0.456, 0.312), 宽=0.823, 高=0.047
# 对应像素坐标：中心=(292, 200), 宽=527 px, 高=30 px
```

**类别映射**：

| YOLO class_id | 类别名 | 内部 4 类对应 |
|:---:|--------|------------|
| 0 | scratch | scratch |
| 1 | spot | spot |
| 2 | critical | damage + crash（合并）|
