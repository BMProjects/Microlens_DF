# 识别算法自动化迭代程序

## 目标

在不影响 GUI、CNAS、评分系统正式主线的前提下，持续提升 `scratch AP@0.5`，并以固定留出测试集的 `mAP@0.5` 作为守门指标。

## 主指标

- Primary: `scratch AP@0.5`
- Guardrail: `mAP@0.5 >= baseline`
- 固定评测入口：`uv run python -m cnas_test.runner.run_eval`

## 允许改动的变量

一次实验只允许改动一类变量：

1. 检测结构：如 `SFE / NWD / P2 head`
2. 训练策略：如学习率、冻结层数、固定训练预算
3. 输入表示：如单通道灰度、灰度+候选图
4. 推理后处理：如 SAHI 参数、IOS / connect_scratches

## 禁止项

- 不修改固定留出测试集
- 不同时修改训练集与验证集定义
- 不把推理后处理回写为训练 GT
- 不直接影响 GUI 默认模型和默认参数

## 单次实验模板

1. 复制父实验配置
2. 只修改一项变量
3. 使用固定预算训练
4. 在固定 `CNAS` 集上评测
5. 记录：
   - 父实验
   - 改动点
   - `scratch AP@0.5`
   - `mAP@0.5`
   - 是否保留
